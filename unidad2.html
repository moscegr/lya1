De Lenguajes Regulares a ER

En la Unidad 1 vimos que los Lenguajes Regulares (Tipo 3) son aquellos que pueden ser procesados sin necesidad de memoria (sin pilas). Las Expresiones Regulares son la herramienta matemática y notacional exacta que utilizamos para describir estos lenguajes. Son el estándar industrial para buscar, extraer y validar información en programación.

2.1

Definición Formal de una ER

Una Expresión Regular sobre un alfabeto \(\Sigma\) se define de manera recursiva. Existen casos base simples y reglas para combinarlos.

Casos Base (Atómicos)

\(\emptyset\)
El lenguaje vacío. No contiene ninguna cadena.
\(\lambda\)
La cadena vacía (a veces representada como \(\epsilon\)).
\(a\)
Cualquier símbolo \(a\) que pertenezca al alfabeto \(\Sigma\).

Notación Matemática

Toda ER denota un lenguaje formal \(L(r)\). Por ejemplo, si nuestra ER es el símbolo \(a\), el lenguaje que representa es un conjunto con un solo elemento:

$$L(a) = \{ a \}$$
2.2

Operaciones (Construcción Inductiva)

Si \(r\) y \(s\) son expresiones regulares que denotan los lenguajes \(L(r)\) y \(L(s)\), podemos combinarlas usando 3 operaciones fundamentales.

1. Unión (\(r + s\) ó \(r | s\))

Acepta cadenas del lenguaje \(r\) O del lenguaje \(s\).

\(L(r+s) = L(r) \cup L(s)\)
Ej: Si \(\Sigma=\{a,b\}\), la ER \(a+b\) denota el lenguaje \(\{a, b\}\).

2. Concatenación (\(rs\))

Una cadena de \(r\) seguida inmediatamente por una de \(s\).

\(L(rs) = L(r) \cdot L(s)\)
Ej: La ER \(ab\) denota exactamente la cadena \(\{ab\}\).

3. Cerradura (\(r^*\))

Cero o más repeticiones de \(r\). (¡Incluye la cadena vacía \(\lambda\)!)

\(L(r^*) = (L(r))^*\)
Ej: \(a^* = \{\lambda, a, aa, aaa, \dots\}\)

Aplicaciones Reales (Regex en Código)

En la programación moderna, la notación formal se adapta usando caracteres ASCII. En lugar del símbolo \(+\), usamos | o corchetes []. El asterisco de Kleene * se mantiene igual.

import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class Validador {
    public static void main(String[] args) {
        // ER matemática: letra(letra+digito)*
        // ER en Java: ^[a-zA-Z][a-zA-Z0-9]*$
        String patron = "^[a-zA-Z][a-zA-Z0-9]*$";
        String entrada = "var123";
        
        // Utilizando la librería nativa
        Pattern p = Pattern.compile(patron);
        Matcher m = p.matcher(entrada);
        
        if (m.matches()) {
            System.out.println("¡Cadena Válida!");
        }
    }
}
