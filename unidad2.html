De Lenguajes Regulares a ER

En la Unidad 1 vimos que los Lenguajes Regulares (Tipo 3) son aquellos que pueden ser procesados sin necesidad de memoria (sin pilas). Las Expresiones Regulares son la herramienta matemática y notacional exacta que utilizamos para describir estos lenguajes. Son el estándar industrial para buscar, extraer y validar información en programación.

2.1

Definición Formal de una ER

Analogía para programadores: Imagina que una Expresión Regular (ER) es como la declaración de un tipo de dato o un "filtro de seguridad". La ER NO es el texto; la ER es la regla que el programa usa para decir: "¿Este String cumple con mis condiciones de seguridad? ¿Sí o no?".

Matemáticamente, para construir cualquier filtro complejo, siempre empezamos con los "ladrillos más pequeños" posibles. A esto se le llama casos base:

Los 3 ladrillos fundamentales

\(\emptyset\)
Conjunto Vacío: Es un filtro imposible de pasar.
En Java: Equivale a una regla rota donde ninguna palabra, nunca, hará "match".
\(\lambda\)
Cadena Vacía: Es un filtro que se pasa "sin escribir nada".
En Java: Equivale a darle "Enter" a un input vacío String s = "";.
\(a\)
Símbolo Terminal: Cualquier letra individual.
En Java: Equivale a exigir explícitamente que te tecleen ese carácter exacto, ej: "a".

¿Qué es L(r)?

En los libros de texto verás mucho la expresión \(L(r)\).

Esto simplemente significa "El Lenguaje generado por la regla 'r'". Es decir, es la lista completa de todas las palabras en el universo que logran pasar el filtro.

Filtro: r = a
L(a) = { "a" }

2.2

Operaciones (Construyendo Filtros)

Solos, los "ladrillos base" no sirven de mucho. Para validar correos o contraseñas, necesitamos pegarlos usando 3 operaciones matemáticas que en programación se traducen a símbolos específicos.

1. Unión (La Alternativa)

Es una estructura "O". Acepta el camino de la izquierda o el de la derecha.

Matemáticas: \(a + b\)
En Java (Regex): a | b

2. Concatenación (La Secuencia)

Es el orden obligatorio. Una cosa escrita exactamente después de la otra.

Matemáticas: \(a \cdot b\) o simplemente \(ab\)
En Java (Regex): ab

3. Cerradura de Kleene (El Bucle)

Repite el elemento anterior desde cero veces hasta el infinito.

Matemáticas: \(a^*\)
En Java (Regex): a*

Construyendo reglas del mundo real

Observa cómo combinando estas 3 simples operaciones podemos validar variaciones complejas de palabras:

color + colour
La Ortografía Dual (Usando Unión): Acepta la palabra escrita en inglés americano (color) O en inglés británico (colour).
L = { "color", "colour" }
ja(ja)*
Detector de Risas (Usando Concat + Kleene): Exige obligatoriamente un "ja" inicial, seguido de cero o infinitas repeticiones adicionales de "(ja)".
L = { "ja", "jaja", "jajaja", "jajajaja"... }
A(0+1)*
Generador de IDs (Usando todo): Toda cadena que empiece estrictamente con la letra "A", seguida de cualquier cantidad y combinación de ceros y unos.
L = { "A", "A0", "A1", "A00", "A0110"... }

El motor en Java: Pattern y Matcher

En Java, validar o buscar texto usando Expresiones Regulares requiere el trabajo en equipo de dos clases principales del paquete java.util.regex:

  • Pattern: Representa la "compilación" de tu expresión regular. Convierte tu string (ej. "\d+") en un Autómata Finito en memoria. Compilar es un proceso "costoso" para la computadora, por lo que se hace una sola vez.
  • Matcher: Es el "cabezal de lectura" que toma el autómata generado por Pattern y lo ejecuta sobre una cadena de texto (String) real, evaluando estado por estado.

Ejemplo 1: Validación Completa (matches)

Usa el método matches() cuando necesitas que toda la cadena encaje exactamente en el patrón, desde el inicio hasta el fin. Útil para validar correos, contraseñas o matrículas.

ValidadorRegex.java

import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class Validador {
    public static void main(String[] args) {
        // ER matemática: letra(letra+digito)*
        // ER en Java: ^[a-zA-Z][a-zA-Z0-9]*$
        String patron = "^[a-zA-Z][a-zA-Z0-9]*$";
        String entrada = "var123";
        
        // 1. Compilar la regla (Pattern)
        Pattern p = Pattern.compile(patron);
        // 2. Aplicar la regla al texto (Matcher)
        Matcher m = p.matcher(entrada);
        
        if (m.matches()) { // Evalúa TODA la cadena
            System.out.println("¡Cadena Válida!");
        }
    }
}

Ejemplo 2: Extracción de Datos (find y group)

Usa el método find() dentro de un bucle `while` cuando quieres buscar múltiples coincidencias ocultas dentro de un texto largo (como extraer todos los números telefónicos de un documento).

ExtractorRegex.java

import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class Extractor {
    public static void main(String[] args) {
        // Texto desordenado con información mezclada
        String ticket = "Producto A: $150.50, Producto B: $20.00";
        
        // Regex: El símbolo de $, seguido de 1 o más dígitos, seguido de . y 2 dígitos
        // Ojo: En Java debemos escapar la barra \\d
        String patron = "\\$\\d+\\.\\d{2}";
        
        Pattern p = Pattern.compile(patron);
        Matcher m = p.matcher(ticket);
        
        System.out.println("Precios encontrados:");
        // m.find() busca la siguiente coincidencia y retorna true si la encuentra
        while (m.find()) {
            // m.group() extrae el pedazo de texto que hizo "match"
            System.out.println("-> " + m.group()); 
        }
    }
}

Puntos a Cubrir (Obligatorios)

Deberás elegir, diseñar y construir una aplicación en Java (consola o interfaz gráfica) que resuelva un problema útil utilizando estrictamente la librería java.util.regex.

  • 1. Uso de la API nativa: El núcleo del programa debe depender de las clases Pattern y Matcher. Queda prohibido el uso de métodos básicos como split(), indexOf() o bucles manuales de caracteres para la lógica principal.
  • 2. Múltiples Operaciones Regex: El proyecto debe implementar al menos dos acciones diferentes del motor. Por ejemplo: Validar un formato (usando matches()) Y extraer información específica (usando find() y group()), o realizar reemplazos estructurados (replaceAll()).

  • 4. Dataset Realista: El programa debe procesar un conjunto de datos lógico. Puede leer un archivo de texto (.txt, .csv, .log) o procesar un bloque grande de texto introducido por el usuario.
